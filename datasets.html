YOI Methodology

Youth Opportunity Index methodology

This page documents how the Youth Opportunity Index (YOI) is constructed in the current dashboard build. The census tract is the base geography. Raw indicators are cleaned at tract level, converted to county-relative percentile scores, reverse-coded when lower values indicate worse conditions, averaged into domain scores, and then equally weighted across seven domains to create the overall YOI.

How the current YOI is built

Base geography
Census tract is the primary analysis level. The main processed output is data/processed/yoi/yoi_components.csv, joined to data/processed/boundaries/sd_tracts.geojson.
Normalization
Each indicator is converted to a percentile rank within San Diego County. Indicators where lower values are worse are reverse-coded so that higher normalized values always mean higher opportunity.
Domain scoring
Indicators are grouped into seven domains: Economic, Education, Health, Housing, Safety / Environment, Mobility / Connectivity, and Youth Supports. The normalized indicators available within each domain are averaged to create the tract-level domain score.
Overall YOI
The current build uses equal domain weights across all seven domains. The overall score is stored as yoi_raw_0_1 and scaled to yoi_0_100 / yoi_custom_0_100 for dashboard display.
Indicator metadata
The frontend explanation panels read data/processed/yoi/yoi_indicator_meta.csv, which stores each indicator’s domain, source label, directionality, and notes used throughout the dashboard.

Geographic levels shown in the dashboard

Census tract
Level of detail: primary analysis geography.
Used where: main choropleth, location details, domain explanations.
Last updated in current build: current processed tract CSV and tract boundary GeoJSON.
ZIP code
Level of detail: aggregated view built from tract-level YOI outputs.
Used where: alternative comparison geography in the dashboard.
How built: tract scores are spatially overlaid with ZIP boundaries and aggregated into data/processed/yoi/yoi_zip_components.csv.
Supervisor district
Level of detail: aggregated policy geography built from tract-level YOI outputs.
Used where: alternative comparison geography in the dashboard.
How built: tract scores are spatially overlaid with supervisor district boundaries and aggregated into data/processed/yoi/yoi_supervisor_district_components.csv.

External reference layer: Child Opportunity Index

What it is
The Child Opportunity Index (COI) is an external benchmark layer used as a comparison overlay, not as part of the YOI calculation.

Link: diversitydatakids.org/child-opportunity-index
How it is used in this dashboard
The dashboard loads a processed San Diego–only COI file, data/processed/overlays/sd_coi_2023.csv. The current pipeline filters COI to San Diego County and to year 2023, then keeps national COI fields such as r_COI_nat and c5_COI_nat.
